### About

I build responsive web interfaces, API-driven features, and maintainable product flows. My core stack is TypeScript, React, Next.js, and Node.js, and I also study C to strengthen my systems and low-level engineering fundamentals.

- Working on frontend features, integrations, and API-connected product flows.
- Building with JavaScript, TypeScript, React, Next.js, Node.js, Docker, and Git.
- Studying C and backend fundamentals to grow beyond UI into stronger engineering depth.
- Improving every week: code quality, communication, and engineering discipline.

### Principles

- Make interfaces understandable before making them fancy.
- Prefer practical architecture over unnecessary complexity.
- Prefer small components with clear ownership.
- Treat frontend and backend contracts as one product surface.
- Keep user states visible: loading, empty, error, and success.
- Write code that another developer can continue without guessing.
